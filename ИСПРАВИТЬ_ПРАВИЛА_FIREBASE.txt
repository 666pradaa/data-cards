ОШИБКА: permission_denied at /users

ПРИЧИНА: Правила Firebase слишком строгие

═══════════════════════════════════════════════════════════════════════════

📍 КАК ИСПРАВИТЬ (2 МИНУТЫ):

1. Откройте: https://console.firebase.google.com/
2. Выберите проект "dota-cards"
3. В левом меню: Realtime Database
4. Вкладка: Rules (вверху)
5. УДАЛИТЕ всё что там есть
6. ВСТАВЬТЕ это:

{
  "rules": {
    "users": {
      ".read": "auth != null",
      ".write": false,
      "$uid": {
        ".write": "$uid === auth.uid"
      }
    }
  }
}

7. Нажмите: Publish (красная кнопка справа вверху)
8. Должно: "Rules published successfully!"

═══════════════════════════════════════════════════════════════════════════

💡 ЧТО ЭТО ДЕЛАЕТ:

".read": "auth != null"  
   → Авторизованные пользователи могут ЧИТАТЬ всех
   
".write": false
   → Никто не может писать в корень users/
   
"$uid": { ".write": "$uid === auth.uid" }
   → Каждый может писать ТОЛЬКО в свой users/{uid}/

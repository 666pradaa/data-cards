ОШИБКА: permission_denied at /users: Client doesn't have permission to access the desired data.

ПРИЧИНА: Firebase Security Rules блокируют доступ к данным

═══════════════════════════════════════════════════════════════
  ✅ РЕШЕНИЕ (ЗАЙМЁТ 2 МИНУТЫ)
═══════════════════════════════════════════════════════════════

ШАГ 1: Откройте Firebase Console
---------------------------------
https://console.firebase.google.com/

ШАГ 2: Выберите ваш проект

ШАГ 3: Realtime Database → Rules
----------------------------------

ШАГ 4: ЗАМЕНИТЕ все правила на это:
------------------------------------

{
  "rules": {
    "users": {
      ".read": "auth != null",
      ".write": "auth != null",
      "$uid": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    },
    "clans": {
      ".read": "auth != null",
      ".write": "auth != null"
    },
    "rooms": {
      ".read": "auth != null",
      ".write": "auth != null"
    }
  }
}

ШАГ 5: Нажмите "PUBLISH" (Опубликовать)
-----------------------------------------

ШАГ 6: Обновите игру (F5)
---------------------------

✅ Теперь вход в аккаунт должен работать!

═══════════════════════════════════════════════════════════════

   • Читать данные всех пользователей (для поиска друзей)

   • Создавать кланы
   • Создавать комнаты

⚠️ Неавторизованные пользователи НЕ ИМЕЮТ доступа
